早期更新博客时，流程比较手动，但每一步都清楚：先改内容和配置，再构建静态文件，最后同步到 Nginx 目录。

## 修改源文件

更新发生在项目目录中，通常涉及：

- `content` 目录
- 站点配置文件
- 主题或样式目录

对于静态博客来说，源文件和发布文件要分清。不要直接把发布目录当成长期维护目录。

## 构建

构建命令：

```bash
hugo
```

构建后的文件进入 `public` 目录。

## 发布

把构建产物复制到 Nginx 目录：

```bash
cp -r public/* /var/www/html/
sudo chown -R www-data:www-data /var/www/html
```

权限处理不能省。否则 Nginx 可能能读一部分文件，却在资源文件或后续覆盖时出问题。

## 新版博客的改进

新的实现会保留“构建后发布静态文件”的方式，但把构建工具换成 Astro。这样页面结构、样式和文章数据都在一个前端项目内管理，后续更适合做高级排版。
