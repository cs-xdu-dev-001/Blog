# 公开页面多设备适配设计

## 目标

让Dev Notes全部公开页面在手机、平板和桌面上保持可读、可操作和一致的视觉气质。管理端不在本次范围内。

## 视口基准

- 手机：390×844
- 平板：768×1024
- 桌面：1440×1000

## 布局原则

- 桌面端现有构图不变。
- 手机端采用单列重排，不把桌面双栏直接压缩。
- 平板端保留适合阅读的双栏，复杂统计和卡片组按空间改为单列或两列。
- 页面不得产生横向滚动，核心文字不低于15px，点击目标不小于40px。
- 首页导航在窄屏保持单行并允许横向滚动，避免文字和社交图标重叠。
- 全站搜索在手机端只显示结果列表，选中结果后进入预览层，可返回结果；平板端保留双栏。
- AI助手在手机端贴合安全边距并占据主要视口，禁用拖动和自由缩放；平板和桌面保留窗口交互。

## 页面范围

- 首页及首页全部模块
- 笔记列表、文章详情
- 书架、书评详情
- 影像档案、影评详情
- About
- 全站搜索和AI助手

## 验收

- 三个基准视口均无横向溢出、遮挡和不可达控件。
- 导航、搜索、AI助手、书架和影像卡片可通过触摸使用。
- 文章、书评和影评保持舒适行宽，代码块和表格允许局部横向滚动。
- `prefers-reduced-motion`继续生效。
