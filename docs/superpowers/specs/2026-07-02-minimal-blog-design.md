# 极简个人博客设计规格

日期：2026-07-02

## 目标

做一个高级感的个人技术博客，视觉上严格走极简主义，不使用 Hugo 主题，也不套通用博客模板。

首版用 Astro 和 TailwindCSS 手写前端，从本地 Notion 笔记中整理 3-5 篇文章，形成可运行、可预览、可继续扩展的静态博客。

## 不做什么

- 不使用 Hugo 或现成博客主题。
- 首版不把 Notion 接成实时 CMS。
- 不做后台、登录、评论、订阅等动态功能。
- 不引入通用 UI 组件库。
- 不做复杂动效，只保留克制的微交互。

## 读者

主要读者是来看技术实践、部署记录、学习笔记的人。网站应优先保证阅读、扫描和可信度，不做营销式首页。

## 内容范围

首版文章从本地 Notion 笔记中整理：

- Hugo 部署
- 网站上线
- 网站更新
- CI/CD 博客
- 爬个人博客

文章不直接复制原始笔记，而是整理成统一的博客格式：

- 标题
- 日期
- 分类
- 简短摘要
- 结构化正文
- 必要的代码块
- 对操作型或不完整内容补充说明

## 页面

### 首页

首页用于介绍作者和这个博客的写作方向，整体像一份排版克制的个人出版物入口。

包含：

- 一行小型元信息
- 大字号主标题
- 简短介绍
- 精选文章
- 最近文章
- 极简页脚

### 文章列表页

文章列表页使用索引式排版，不做卡片堆叠。

每一行包含：

- 编号
- 日期
- 分类
- 标题
- 简短摘要

行与行之间用细分隔线和网格对齐，不使用阴影。

### 文章详情页

文章详情页以阅读为核心。

包含：

- 标题
- 日期和分类
- 可选摘要
- 正文
- 代码块样式
- 返回文章列表的链接

桌面端阅读宽度控制在 720-800px 左右。

### 关于页

关于页保持简短。

包含：

- 这个网站写给谁看
- 当前关注方向
- GitHub、邮箱或 RSS 等真实链接；没有实际价值的链接先隐藏

## 视觉方向

视觉系统遵循你给的 Minimalism UI 参考：

- 白色背景
- 黑色主文字
- 灰色建立信息层级
- 不使用装饰渐变、阴影、色块、光斑或插画
- 依靠排版、留白和网格形成视觉秩序
- 点缀色只用于链接或焦点状态

## 布局系统

桌面端使用 12 列网格：

- `grid-template-columns: repeat(12, 1fr)`
- 桌面端列间距 32px
- 平板和移动端根据空间收窄到 24px 或更低
- 桌面端内容占页面宽度约 40-50%
- 留白占页面宽度约 50-60%

间距使用 8px 基础单位：

- 8px
- 16px
- 24px
- 32px
- 48px
- 64px
- 96px

页面主区域留白：

- 桌面端按场景使用 64px
- 移动端压缩到 24-32px

## 排版

字体栈：

```css
Inter, "Helvetica Neue", "SF Pro Display", Arial, sans-serif
```

字号层级：

- 首页主标题：48-72px，字重 300 或 700，字距 -0.02em
- 区块标题：24-36px，字重 400
- 正文：16-18px，行高 1.8，字距 0.02em
- 辅助文字：12-14px，浅字重，灰色

排版本身就是主要视觉，不用大面积卡片和装饰表面。

## 颜色

主色：

- 黑色：`#000000`
- 白色：`#ffffff`

文字层级：

- `#1a1a1a`
- `#2d2d2d`
- `#404040`
- `#666666`
- `#808080`
- `#999999`

线条和浅色表面：

- `#cccccc`
- `#e5e5e5`
- `#f5f5f5`

点缀色：

- 链接蓝：`#0066ff`，少量使用
- 红色：`#ff0000`，后续如有警告或删除状态再使用

## 交互

交互保持克制：

- 链接 hover 改变颜色或透明度。
- 文章行 hover 改变边框颜色或文字颜色。
- 按钮可以做黑白反转。
- 过渡使用 `transition: all 0.3s ease`。
- 不使用缩放、弹跳、视差、大位移或滚动劫持。

键盘焦点状态必须可见。

## 技术架构

使用：

- Astro
- TailwindCSS
- TypeScript
- Markdown 或 MDX 内容文件
- Astro Content Collections 管理文章元数据

建议目录：

```text
src/
  components/
    Header.astro
    ArticleIndex.astro
    MinimalFooter.astro
  layouts/
    BaseLayout.astro
    ArticleLayout.astro
  pages/
    index.astro
    writing.astro
    about.astro
    posts/[slug].astro
  content/
    posts/
      hugo-deploy.md
      website-launch.md
      website-update.md
      ci-cd-blog.md
      crawl-personal-blog.md
  styles/
    global.css
```

## 组件边界

`BaseLayout.astro`

- HTML 外壳
- 通用元信息
- 全局头部和页脚
- 主容器结构

`ArticleLayout.astro`

- 文章标题区
- 文章元信息
- Markdown 正文样式
- 返回导航

`Header.astro`

- 极简文字导航
- 链接：Home、Writing、About

`ArticleIndex.astro`

- 可复用文章列表
- 支持精选列表和完整索引两种形态

`MinimalFooter.astro`

- 版权信息
- 少量链接

## 验证

实现完成前需要验证：

- 启动本地开发服务器。
- 检查首页、文章列表页、关于页和至少一篇文章页。
- 运行生产构建。
- 用浏览器截图检查桌面端和移动端排版。
- 确认移动端文本不溢出、不重叠、不塌陷。
- 确认整体仍然是极简主义：没有阴影、装饰渐变、组件库默认样式。

## 已确认决策

首版做一个 polished static blog：静态博客、本地 Markdown 内容、极简视觉、轻量微交互。
