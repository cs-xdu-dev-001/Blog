# 影像档案管理端设计规格

日期：2026-07-03

## 目标

把影像档案从“开发者手工改文件”升级为“服务器上的内容管理端”。用户以后访问 `/admin`，像商家管理商品一样维护电影和电视剧条目：写评论、填佳句、上传对应图片、保存后前台即时更新。

首版只做影像档案管理，不做文章编辑、阅读档案、站点配置。这样可以先把最常用、最有价值的链路做扎实。

## 用户场景

用户打开后台后，可以完成这些事：

1. 查看所有影视条目。
2. 筛选缺图、缺评论、缺佳句、已看、想看、精选。
3. 搜索影视名。
4. 编辑某个影视条目的个人评论、佳句、来源、评分、状态、精选标记。
5. 上传该影视对应图片，默认保存为 `影视名.扩展名`。
6. 保存后首页影像档案立即使用新内容和新图片。

## 非目标

首版不做这些：

- 多用户协作。
- 复杂权限系统。
- 在线文章 Markdown 编辑器。
- 阅读档案管理。
- 站点全局配置管理。
- 图片裁剪、AI 修图、批量素材库。
- 复杂数据分析报表。

这些可以在影像管理端稳定后再加。

## 技术方向

采用动态博客模式：

- Astro SSR。
- SQLite 数据库。
- `/admin/login` 登录页。
- `/admin/watch` 影像管理页。
- `/api/admin/watch/*` 管理接口。
- 上传图片保存到服务器本地目录，例如 `public/uploads/watch/`。
- 前台影像模块从数据库读取数据。

保存内容后不需要重新 build，前台刷新即可看到新内容。

## 数据模型

### `watch_items`

字段：

- `id`：主键。
- `title`：影视名，必填，唯一优先，但允许人工处理重名。
- `type`：`电影` / `剧集` / `待看`。
- `status`：`已看` / `想看`。
- `rating`：评分，可空。
- `comment`：个人评论，可空。
- `quote`：佳句，可空。
- `quote_source`：佳句来源，可空。
- `image_path`：图片路径，可空。
- `is_featured`：是否精选。
- `created_at`：创建时间。
- `updated_at`：更新时间。

### 图片命名规则

默认规则：

`public/uploads/watch/影视名.jpg`

上传时：

- 如果用户上传 `jpg/png/webp/avif`，服务端按影视名重命名。
- 如果同名文件已存在，覆盖旧图。
- 如果影视名包含 Windows/Linux 文件名不适合字符，服务端做安全转义，同时数据库保存最终路径。
- 前台只读取 `image_path`，不再猜测多个目录。

## 管理端页面结构

视觉方向：darkMode 内容控制台，参考开发者仪表板，不做普通白底表格后台。

布局：

- 顶部栏：`Dev Notes CMS`、搜索框、保存状态、预览前台。
- 左侧栏：影像档案、阅读档案占位、文章草稿占位、站点配置占位。
- 中间区：
  - 指标卡：总条目、缺图、缺评论、缺佳句。
  - 筛选 chips：全部、缺图、缺评论、缺佳句、已看、想看、精选。
  - 影视条目列表。
- 右侧编辑区：
  - 当前图片预览。
  - 上传图片按钮。
  - 状态、评分、精选开关。
  - 个人评论 textarea。
  - 佳句 textarea。
  - 佳句来源 input。
  - 保存按钮。

## 交互规则

- 点击列表条目后，右侧编辑区加载该条目。
- 修改字段后显示“未保存”状态。
- 点击保存后调用 API 更新数据库。
- 上传图片后立即显示预览，但仍需要保存条目以确认。
- 保存成功后显示“已保存”，前台刷新可见。
- 缺失项用文字 + 状态标签提示，不只靠颜色。
- 表单支持键盘访问和清晰 focus ring。

## 登录与安全

首版做单管理员登录：

- 环境变量配置管理员用户名和密码哈希。
- 登录成功后写入 HTTP-only session cookie。
- `/admin/*` 页面和 `/api/admin/*` 接口都要求登录。
- 上传接口限制文件类型和大小。
- API 不接受任意路径写入，只能写入指定上传目录。

## 数据迁移

初始迁移从当前 `src/data/watchItems.generated.mjs`、`watchLines.mjs`、`watchImages.mjs` 导入。

导入规则：

- 标题、类型、状态、评分来自现有 generated 数据。
- 已有佳句来自 `watchLines.mjs`。
- 已有图片路径来自 `watchImages.mjs` 或已有本地图片。
- 如果没有评论，留空，不自动生成。

## 前台同步

首页影像档案改为读取数据库数据：

- 有图片时显示上传图片。
- 有佳句时显示佳句。
- 有评论但无佳句时可显示评论摘要。
- 两者都没有时显示安静的档案状态文案。

前台保持当前双排影像轨道和卡片视觉，不重新设计首页。

## 验证标准

实现完成后必须验证：

- 登录保护生效。
- 能搜索和筛选影像条目。
- 能保存评论、佳句、评分、精选状态。
- 能上传图片，并按影视名匹配。
- 前台首页能显示新图片和新文案。
- 桌面端和移动端后台没有明显溢出。
- `npm run build` 或 SSR 启动检查通过。

## 首版交付边界

首版完成后，用户应能在服务器上通过浏览器维护影像档案，不再需要 SSH 修改影视内容文件。

阅读档案、文章编辑、站点配置作为后续模块，沿用同一套后台框架扩展。
